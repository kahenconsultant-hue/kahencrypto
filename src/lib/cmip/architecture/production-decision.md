# CMIP Production Decision Boundary

Version: CMIP-PRODUCTION-DECISION-BOUNDARY-1.0

Full canonical CMIP Morning Brief report generation by OpenAI or Gemini is experimental. It is not approved as the primary production report-generation path.

Production canonical report structure must be generated by deterministic application code. AI may not own numerical engine scores, decision posture, confidence numbers, scenario probabilities, ten-asset identity, source references, calculation references, audit records, or final canonical assembly.

The approved production direction is:

1. Collect and normalize runtime input deterministically.
2. Calculate engines, source/calc references, evidence status, confidence caps, scenario constraints and audit records deterministically.
3. Assemble the Task 001 canonical report with application code.
4. Run the unchanged Task 001 schema and semantic validators.
5. Use AI only for future bounded explanatory text based on locked values, after a separate approved contract.

The current full-report AI paths remain available only for explicit admin testing, dry runs, compatibility tests, controlled research, and future provider comparison. They require the server-side `CMIP_ENABLE_EXPERIMENTAL_FULL_REPORT_AI=true` capability gate for non-fake execution and must never be selected silently by production code.

The reserved future task type `explanation_only` is not implemented in this task. It exists only to name the intended bounded explanation layer.
