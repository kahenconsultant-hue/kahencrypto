# Analogy Model

Analogy is separate from statistical historical evidence. It is descriptive and must not become a forecast.

Every analogy output must include:

- `analogue_id`
- `historical_period`
- `similarity_score`
- `similar_features`
- `different_features`
- `regime_match`
- `outcome_after_1d`
- `outcome_after_7d`
- `outcome_after_30d`
- `limitations`

Rules:

- Analogy is descriptive.
- Analogy must not become a forecast.
- Similarity score must be method-based.
- False similarity risk must be explained.
- Structural differences must be shown before outcome comparison.
- Unavailable analogue data must remain null.

Simple Persian explanation must say:

> این فقط شباهت تاریخی است، نه پیش‌بینی تکرار همان نتیجه.
