# Conflict Resolution Model

Conflict is a first-class state. It must be detected, preserved, explained, and audited.

## Conflict Types

- `source_conflict`
- `timeframe_conflict`
- `domain_conflict`
- `identity_conflict`
- `calculation_conflict`
- `historical_conflict`
- `decision_memory_conflict`

## Rules

1. Never average conflicting values silently.
2. Prefer authoritative primary sources, but preserve weaker conflicting values when unresolved.
3. Preserve both values when resolution is not justified.
4. Reduce confidence when conflict remains.
5. Block a field from scoring when identity is unresolved.
6. Distinguish short-term positive from medium-term negative.
7. Explain material conflicts in simple Persian.

## Output Requirement

The final decision must include unresolved material conflicts, the affected fields, why the final posture still prevailed, and what would resolve the conflict.

Example simple Persian:

> داده‌های کوتاه‌مدت مثبت است، اما داده‌های ۳۰ روزه هنوز ضعف نشان می‌دهد. برای همین نتیجه قطعی نیست و اعتماد مدل پایین‌تر می‌آید.
